WordPress Features

* A robust back-end for entering content
* Content geared toward posts and pages
* A front-end for representing your content
* A templating engine for creating your own themes
* Numerous free themes
* Open Source
* Self Hosted & Hosted Solutions
* Powered by PHP and MySQL
* Thousands of plugins
* Deployment ready out of the box

WordPress Challenges

  • The templating engine isn't the best
  • Predefined Tech Stack
  • Theme and Plugin Security
  • Deep customization requires extensive experience
  • The more features that are layered on, the more complex the task

WordPress...

  • is at it's heart, a blogging platform
  • is great for rapidly developing a website
  • requires little technical expertise
  • can handle multiple content contributers with ease
  • offers many free themes and plugins
  • is great for small business

What is Contentful?

Contentful is a CMS without a front-end

An empty Contentful space. A space is the enviroment you are working in.

From within Contentful, you define what data you need to store.

Example Contentful Workflow

1. Define your content model by creating content types
2. Input your content by creating instances of content types
3. Sync your content with your web app using a Contentful SDK

There are many different kinds of fields you can add to a content type.

You also have numerous options for validating those fields.

An example of a finished piece of content. We can create multiple instances of each content type. The same way one might create multiple blog posts.

Contentful Features

* Contentful does not have a front-end
* Develop apps on your favorite stack (including the database)
* Contentful manages and delivers your content
* API centric, get data via API calls
* JSON delivery of data without plugins
* Contentful WebApp allows content editors to use a feature rich / user friendly system for editing content
* SDKs for many languages (Ruby, Android, iOS, PHP, Java, .Net, Javascript, Python)
* Hosted solution
* Built for creating any content
* Webhooks for Content Delivery

Contentful Challenges

  • Requires a developer
  • Requires a designer for the front-end
  • Not ready 'out of the box'
  • Steeper learning curve than WordPress (for developers)

Contentful....

  • is unopinionated about your technology stack
  • gives your content editors a rich CMS to define and enter content
  • has easy to use APIs for content delivery
  • offers the freedom to develop decoupled, from the constraints of a CMS

Sample Scenarios

Scenario A

A small business website that is mostly static, with the addition of a contact form and simple blog features.

WordPress: WordPress is the great solution for a project like this. It would be easy to setup the website, pick a free theme, enter the content and install a contact form plugin. No coding required.
Contentful: Overkill for this project.

Scenario B

A startup is developing a new website built on React with an Express back-end, it will have informational pages which are expected to be updated frequently as well as a product blog. They need a CMS to update all aspects of the content that is presented on the website, including images and files.

WordPress: Could we do this with WordPress? Maybe. Would we want to? Probably not.
Contentful: Instead of building a fully featured CMS, or customizing an out of the box CMS solution for the back-end, Contentful can be used. The front-end and back-end can be developed decoupled from the CMS and without a large amount of resources spent developing a CMS from scratch.

Scenario C

A business is looking to build a multilingual (english and frensh) site. Not only does the content need to be localized, but the images and files as well. They are also interested in this being a single page app.

Secnario C

WordPress: Using specific themes and plugins it would be possible to localize the content fully across multiple langages. The single page aspect is harder, because of the way WordPress templating works and while it might be possible, it would be messy.

Contentful: Let's say we use Vue.js on the front-end and the Laravel framework on the back-end with the PHP SDK for Contentful you could easily integrate your CMS solution with your stack. Contentful offers language localiation features on a per field basis, meaning you can localize one field of a piece of content but not another. It also allows you to localize images and PDFs easily.

Conclusion

Both WordPress and Contentful have excellent use cases. Which you choose really depends on the scope and requirements of the project.
